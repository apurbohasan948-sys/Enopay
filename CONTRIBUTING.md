# Contributing to SMS TX Organizer

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to SMS TX Organizer. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
-   Use the GitHub Issues tab.
-   Include a clear descriptive title.
-   Provide steps to reproduce the issue.
-   Mention the Android version and device you are using.

### Suggesting Enhancements
-   Explain why this enhancement would be useful.
-   Provide mockups or code snippets if possible.

### Pull Requests
-   Ensure your code follows the existing style (Kotlin for Android, TypeScript for Web).
-   Make sure your changes don't break existing functionality.
-   Keep pull requests small and focused on a single change.

## Development Setup

### Android
1.  Open the `android/` directory in Android Studio.
2.  Use the Kotlin Style Guide.
3.  Ensure `google-services.json` is not committed.

### Web Dashboard
1.  Run `npm install`.
2.  Use `npm run dev` to start the development server.
3.  Follow the Tailwind CSS guidelines for styling.

## Questions?
If you have any questions, feel free to open an issue or reach out to the project maintainer.

Happy coding!
